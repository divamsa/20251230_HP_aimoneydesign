# 株式会社マネーデザイン 生成AI導入支援サービス HP 要件定義書
## ai.moneydesign.co.jp プロジェクト

## 📋 プロジェクト概要

### プロジェクト名
株式会社マネーデザイン 生成AI導入支援サービス 公式Webサイト

### 目的
生成AI導入支援サービス（中小企業、地方自治体、博物館、美術館など）の専用HPを作成し、サービス認知度向上と問い合わせ獲得を目指す。

### 現状の課題
- FP✖️不動産✖️生成AIを謳っているが、現在は生成AI導入支援に全振りしている
- 生成AI導入支援サービス専用のHPが存在しない
- サービス内容や実績を適切に伝える場がない

### ドメイン・サーバー情報

**⚠️ 重要: 方針変更（2025年12月29日）**

- **既存ドメイン**: https://moneydesign.co.jp
- **サブドメイン**: **ai.moneydesign.co.jp** ✅ 決定
- **本番サーバー**: **X Server** ✅ 決定
- **開発環境**: ローカル環境（Mac）でLaravel開発
- **構成**: `/studio` とは別の独立したLaravelベースのHPとして構築
- **既存HPとの関係**: 相互リンクを設置し、ブランド統一を図る
- **プロジェクトの位置づけ**: 実験的なLaravelベースのHPとして、X Serverに新規で構築

### 開発方針の変更理由

- `/studio` とは別に、LaravelベースのHPを実験的に作成
- これまで学んだLaravelの知識を実践的に活用
- X Serverに新たに乗せて、独立したサービスサイトとして運用
- `/studio` の既存HPとは別のプロジェクトとして進める

### サブドメイン選定の検討事項
- **SEO**: 短く覚えやすいドメイン名
- **ブランディング**: サービス内容が伝わる名前
- **既存HPとの関係**: 統一感を保ちつつ、独立したサービスであることを示す
- **将来的な拡張**: 他のサービスもサブドメインで展開する可能性

### サイトマップ（全体構成）

#### ページ数の整理

**メインページ: 6ページ**

1. **トップページ（/）** - 1ページ
2. **サービス紹介（/services）** - 1ページ（3つのサービスを1ページにまとめる）
3. **導入事例・実績（/cases）** - 1ページ（業種別にフィルターで表示）
4. **会社情報（/about）** - 1ページ
5. **お問い合わせ（/contact）** - 1ページ（3つのフォームを1ページにまとめる）
6. **ブログ・お知らせ（/blog）** - 1ページ + 記事詳細ページ（動的）

**合計: 6ページ（ブログ記事除く）**

#### 詳細なページ構成

**オプション1: シンプル構成（推奨）**
```
/
├── /services（サービス紹介） - 1ページに全サービスを記載
├── /cases（導入事例・実績） - 1ページに全事例を記載（フィルター機能）
├── /about（会社情報）
├── /contact（お問い合わせ） - 1ページに全フォームを記載
└── /blog（ブログ）
    └── /blog/{id}（記事詳細） - 動的ページ
```

**オプション2: 詳細構成（将来的な拡張用）**
```
/
├── /services（サービス紹介）
│   ├── /services/consulting（コンサルティング詳細）
│   ├── /services/solution（ソリューション詳細）
│   └── /services/training（研修・トレーニング詳細）
├── /cases（導入事例・実績）
│   ├── /cases/small-business（中小企業事例一覧）
│   ├── /cases/local-government（地方自治体事例一覧）
│   └── /cases/museum（博物館・美術館事例一覧）
├── /about（会社情報）
├── /contact（お問い合わせ）
│   ├── /contact/inquiry（一般問い合わせ）
│   ├── /contact/download（資料請求）
│   └── /contact/consultation（無料相談）
└── /blog（ブログ）
    └── /blog/{id}（記事詳細）
```

**推奨: オプション1（シンプル構成）**
- 初期は6ページでスタート
- 必要に応じてサブページを追加可能
- 開発コストと運用コストを抑えられる
